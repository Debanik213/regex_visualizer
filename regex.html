Understanding Regular Expressions

The Regex Builder works by taking a regular expression as input and using it to generate a set of matching strings. It’s like creating a unique blueprint for patterns in text.

Σ The Alphabet

The set of all possible characters you can use. For example: {a, b}.

ε Empty String

Represents a string with no characters. It's the 'zero' of regular languages.

token Operators

  • Union (|): Choice between two patterns.
  • Star (*): Zero or more repetitions.
  • Plus (+): One or more repetitions.

list_alt Semantic Operator Palette

  • * (Kleene Star): Zero or more occurrences.
  • + (Kleene Plus): One or more occurrences.
  • ? (Optional): Zero or one occurrence.
  • | (Union): The logical "OR" operator.
  • ( ) (Capture Group): Groups tokens together.
  • [ ] (Character Class): Matches any single character within.
  • ^ / $ (Anchors): Specify start or end of string.
  • ε (Epsilon): The empty string symbol.
  • Ø (Empty Set): The null set symbol.
